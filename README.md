# VoyagePilot AI - Smart Travel Planner

Production-quality **Agentic RAG Travel Planner** with a premium animated UI, live tool integrations, and grounded travel recommendations.

## What This Project Does

VoyagePilot AI combines:
- **RAG (Retrieval-Augmented Generation)** over travel sources
- **Agent planning + tool execution** (LangGraph)
- **Live weather + news tools**
- **Auto-ingestion for unseen destinations**
- **Conversation memory + follow-up suggestions**
- **Immersive animated frontend** (map-like route visualization)

You can ask for a trip plan, then continue with up to 9 follow-up questions in the UI.

---

## Key Features

- Agent workflow: **Thought -> Action -> Observation -> Final Answer**
- Weather consent handling + inline weather approval in query text
- First answer auto-enriched with weather (when location is available)
- Nearby-location fallback when exact weather location is not found
- News lookup for safety/issues/news intent
- FAISS vector store with metadata-aware retrieval
- Crawl pipeline with fallback scraping behavior on Windows
- Citations embedded in final answer body
- Follow-up question suggestions generated by LLM
- Stylish animated UI with:
  - pulsing nodes
  - animated travel routes
  - particles and parallax depth

## Setup

### 1) Clone and create venv

```powershell
git clone <your-repo-url>
cd <repo-folder>
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 2) Install dependencies

```powershell
pip install -r requirements.txt
```

### 3) Configure environment

```powershell
Copy-Item .env.example .env
```

Set these values in `.env`:

- `OPENAI_API_KEY`
- `OPENWEATHER_API_KEY`
- `NEWS_API_KEY`
- `OPENAI_MODEL` (default: `gpt-4o-mini`)
- `OPENAI_EMBEDDING_MODEL` (default: `text-embedding-3-small`)
- `FAISS_INDEX_DIR` (default: `backend/data/faiss_store`)
- `LOG_LEVEL` (default: `INFO`)
- `CRAWL4AI_ENABLED` (default false on Windows)

### 4) Optional Playwright install (only if enabling crawl4ai browser path)

```powershell
python -m playwright install
```

## Run the App

```powershell
python -m uvicorn backend.api.main:app --reload --port 8000
```

Open:
- App UI: `http://127.0.0.1:8000/`
- API docs: `http://127.0.0.1:8000/docs`
- Health: `http://127.0.0.1:8000/health`

## API

### POST `/query`

Request:

```json
{
  "query": "Plan a budget-friendly trip to Kashmir with weather details",
  "location": "Kashmir"
}
```

Response:

```json
{
  "answer": "Thought: ... Action: ... Observation: ... Final Answer: ... Citations: ...",
  "sources": ["https://..."],
  "actions_taken": ["weather_tool:auto_first_answer"],
  "suggested_followups": [
    "What should I book first?",
    "Can you optimize this for lower cost?",
    "What safety tips should I keep in mind?"
  ]
}
```

## Ingestion + Retrieval Notes

- If `location` is not found in FAISS, ingestion is auto-triggered.
- Ingestion builds chunks with metadata:
  - `location`
  - `category`
  - `source`
  - `source_url`
- Retrieval is semantic and location-aware, with fallback broad retrieval.

---

## Weather and Location Behavior

- First answer automatically includes weather when location is present.
- If exact location lookup fails, weather tool tries nearby candidates.
- If a nearby location is used, the answer includes a note.

## Known Windows Notes

- Some Windows setups can fail Playwright subprocess calls.
- This project includes fallback scraping via `requests` when crawl4ai path is unavailable.
- Keep `CRAWL4AI_ENABLED=false` unless you specifically need browser extraction and your setup supports it.

---

## Security & Ops Recommendations

- Never commit `.env` to Git.
- Rotate API keys immediately if exposed in logs.
- Consider adding rate limits/auth before public deployment.
